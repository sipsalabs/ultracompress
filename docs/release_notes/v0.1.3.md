# UltraCompress v0.1.3

**Release date**: 2026-04-29
**Type**: Minor / non-behavioral
**Upgrade urgency**: Low — recommended for any installation that goes through enterprise procurement, federal customer review, or EU customer onboarding. No CLI behavior change vs v0.1.2.

## TL;DR

Open-source compliance hardening pass. Adds a third-party license attribution file and a CycloneDX 1.6 Software Bill of Materials (SBOM) to the wheel and source distribution. Aligns the published distribution with the EU Cyber Resilience Act (effective 2027), US Executive Order 14028, and standard enterprise security questionnaire expectations. No CLI behavior changes; `uc list`, `uc pull`, `uc info`, `uc bench`, and `uc demo` are byte-for-byte equivalent to v0.1.2.

## Added

### `THIRD_PARTY_LICENSES.txt`

Comprehensive attribution document for every runtime dependency:

| Package | Version constraint | License |
| --- | --- | --- |
| huggingface_hub | >= 0.24.0 | Apache License, Version 2.0 |
| safetensors | >= 0.4.0 | Apache License, Version 2.0 |
| tqdm | >= 4.66.0 | MPL-2.0 AND MIT |
| click | >= 8.1.0 | BSD-3-Clause |
| rich | >= 13.0.0 | MIT |
| PyYAML | >= 6.0 | MIT |

The file ships at the wheel's `ultracompress_cli/THIRD_PARTY_LICENSES.txt` path and at the repository root. It includes the full unmodified text of each license family used (Apache 2.0, BSD-3-Clause, MIT, and MPL-2.0).

### `SBOM-cyclonedx.json`

CycloneDX 1.6 Software Bill of Materials describing the runtime dependency graph with PURLs and distribution URLs. Generated via the `cyclonedx-py requirements` tool. Ships at `ultracompress_cli/SBOM-cyclonedx.json` inside the wheel and at the repository root.

This is the SBOM customers will increasingly be required to attach to procurement diligence, security questionnaires (SIG, CAIQ), and federal-customer compliance checks.

## Changed

### Build configuration

`pyproject.toml` now includes:

- `[tool.hatch.build.targets.wheel.force-include]` — bundles `THIRD_PARTY_LICENSES.txt` and `SBOM-cyclonedx.json` into the wheel
- `[tool.hatch.build.targets.sdist].include` — ensures both files plus `CHANGELOG.md`, `CONTRIBUTING.md`, `CODE_OF_CONDUCT.md`, and `SECURITY.md` are part of the source distribution

### Version strings

`__version__` updated to `0.1.3` in `src/ultracompress_cli/__init__.py`. The `uc demo` install scene now displays `v0.1.3`. No other source changes.

## Unchanged

- CLI surface: `uc list`, `uc pull`, `uc info`, `uc bench`, `uc demo`, `uc version`
- Runtime dependencies and version pin ranges
- Apache-2.0 license terms
- Manifest schema, model artifact compatibility, and Hugging Face Hub integration

## Patent posture

UltraCompress is patent pending. Two USPTO provisional applications were filed on 2026-04-25:

- USPTO 64/049,511 (Row-Overlay Quantization (RoQ), shipping in this release)
- USPTO 64/049,517 (Fractal Residual Recursion (FRR), research)

This release adds OSS-compliance artifacts only and does not change the patent claim scope, the published methodology, or the prior art differentiation strategy.

## Upgrade

```
pip install --upgrade ultracompress
```

Or pin explicitly:

```
pip install "ultracompress==0.1.3"
```

Verify the upgrade:

```
uc --version          # uc, version 0.1.3
uc list               # current Hugging Face Hub catalog state
```
