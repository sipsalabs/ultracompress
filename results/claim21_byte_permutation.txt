INTERPRETATION
--------------
- gap_pp near 0 => the coder is already operating at the order-0 Shannon
  limit of that stream. Byte permutation cannot hurt a coder that
  only uses marginal byte statistics.
- gap_pp > 0   => the coder is finding higher-order context (e.g., run
  length, repeated subsequences, delta patterns). Byte permutation
  destroys this structure while preserving the histogram.
- Histogram equality between orig and perm is asserted at encode time
  (np.bincount match) so the order-0 distribution is provably identical.
